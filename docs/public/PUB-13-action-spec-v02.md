# 13 · 跟进建议 JSON 规格（v0.2）

> v0.1 五字段扁平 JSON 适合 POC；v0.2 要求 **可审批、可执行、可核对查证**。  
> 输入侧：`prompt_user` 已含生产级「系统查证」；输出侧须 **结构化回应查证**，而非复述一句「催签约」。

## 设计原则（参考结构化输出实践）

| 原则 | 说明 |
|------|------|
| **分层** | 决策 → 情况判断 → 行动方案 → 引用查证 |
| **可核对** | `引用查证` 每条必须能在用户消息「系统查证」中找到对应事实 |
| **可执行** | `沟通要点` 2～4 条，像管家电话提纲，不是空话 |
| **克制** | 不新增查证未出现的金额、部位、渠道、合同状态 |
| **中文** | 键名与枚举一律中文 |

## v0.2 JSON 结构

```json
{
  "规格版本": "v0.2",
  "需要跟进": true,
  "优先级": "中",
  "客户情绪": "中性",
  "原因摘要": "待签约停留17天，已报价6598元未支付，窗户方案，建议今日电话推进签约。",
  "优先级依据": [
    "停留17天，接近14天跟进窗上限",
    "正式报价6598元仍未支付",
    "渗漏部位为窗户，非屋面超高客单"
  ],
  "情况判断": {
    "商机阶段": "待签约",
    "报价状态": "已正式报价未签约",
    "金额与方案": "6598元；X3有保温窗户（小）等",
    "渠道与部位": "京东旗舰店；窗户"
  },
  "跟进方案": {
    "主行动": "电话确认报价并推进签约",
    "沟通要点": [
      "确认客户是否看清6598方案与质保3年",
      "了解是否在比价或装修进度影响签约",
      "明确下一步签约时间与付款方式"
    ],
    "避免事项": [
      "勿承诺未在查证出现的折扣或工期"
    ]
  },
  "引用查证": [
    "正式报价6598元（未支付）",
    "获客渠道：电商/京东/东方雨虹官方旗舰店",
    "渗漏部位：窗户"
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规格版本 | string | 是 | 固定 `"v0.2"` |
| 需要跟进 | bool | 是 | 是否建议管家投入精力 |
| 优先级 | 高/中/低 | 是 | 结合停留、金额、部位、渠道、业务提示 |
| 客户情绪 | 积极/中性/消极 | 是 | 基于备注/流程，无则中性 |
| 原因摘要 | string | 是 | 1～2 句，给管家扫一眼 |
| 优先级依据 | string[] | 是 | 2～4 条，短句 |
| 情况判断 | object | 是 | 对查证的归纳，非新事实 |
| 跟进方案 | object | 是 | 主行动 + 沟通要点 + 避免事项 |
| 引用查证 | string[] | 是 | 2～5 条，摘自系统查证 |

### 报价状态枚举（情况判断）

- `已正式报价未签约`
- `无正式报价`
- `已有生效签约`（此时通常 `需要跟进=false`）

## 与 v0.1 兼容

解析层仍接受 v0.1 扁平字段（`原因`、`建议动作`），自动映射为 v0.2 结构。
